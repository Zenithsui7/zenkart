You are a senior full-stack product engineer and UI/UX architect. Your task is to build "ZenKart" — a next-generation super e-commerce platform that combines the best features of Amazon, Flipkart, Meesho, and OLX into one unified app.

---

## 🏷️ APP IDENTITY
- Name: ZenKart
- Tagline: "Shop. Sell. Discover. Everything, Everywhere."
- Brand Colors: Deep Indigo (#1a1a2e), Electric Teal (#00d4aa), Warm Amber (#ffb347)
- Logo: A stylized "Z" merging a shopping cart and a leaf (modern, trustworthy, vibrant)
- Font Style: Clean, modern — think Poppins for headings, Inter for body
- Target Market: India (support ₹ currency, Hindi/English toggle, UPI/COD payments)

---

## 🔐 AUTH & USER SYSTEM
- OTP-based login via mobile number (like Meesho/Flipkart)
- Email + password login
- Google / Facebook OAuth
- Role-based accounts:
  - Buyer (default)
  - Seller (apply to become one)
  - Reseller (like Meesho — share products and earn commission)
  - Used Goods Seller (like OLX — list second-hand items)
- Profile: avatar, address book, wallet, order history, wishlist, reviews

## 🗄️ TECH STACK (Recommended)
- Frontend: React Native (mobile) + Next.js (web)
- Backend: Node.js + Express OR Django REST Framework
- Database: PostgreSQL (main) + Redis (cache/sessions)
- Search: Elasticsearch or Algolia
- Storage: AWS S3 (images/videos)
- Auth: Firebase Auth or custom JWT
- Payments: Razorpay (supports UPI, cards, COD)
- Real-time chat: Socket.io
- Notifications: Firebase Cloud Messaging (FCM)
- Maps: Google Maps API (for OLX-style location listings)
- AI/ML: Python microservice for recommendations

---

## 🧱 DATABASE SCHEMA (Core Tables)
- users (id, name, phone, email, role, wallet_balance, created_at)
- products (id, title, description, price, mrp, images, category, seller_id, stock, rating)
- orders (id, user_id, items_json, total, status, address_id, payment_method)
- listings (id, user_id, title, description, price, images, condition, location, is_negotiable) — OLX listings
- reseller_shares (id, reseller_id, product_id, share_price, commission_earned)
- chats (id, buyer_id, seller_id, listing_id, messages_json)
- reviews (id, product_id, user_id, rating, text, images)
- coupons (id, code, discount_type, value, min_order, expiry)
- wallet_transactions (id, user_id, amount, type, reference)
- notifications (id, user_id, title, body, type, read, created_at)

## 🎨 UI/UX RULES
- Mobile-first design (320px to 428px primary)
- Bottom navigation bar: Home | Categories | Sell | Orders | Profile
- Skeleton loaders (no blank loading screens)
- Haptic feedback on add-to-cart, checkout
- Dark mode support
- Swipeable product image carousels
- Sticky "Add to Cart" + "Buy Now" buttons on product page
- Toast notifications for actions (added to cart, wishlist, etc.)
- Pull-to-refresh on all listing screens

---

## ✅ QUALITY REQUIREMENTS
- All screens must be pixel-perfect and production-ready
- No placeholder "Lorem ipsum" — use realistic Indian product data
- Prices in ₹ (INR), distances in km
- Error states, empty states, and loading states for every screen
- Accessibility: minimum AA contrast ratio, touch targets ≥ 44px
- Performance: images lazy-loaded, lists virtualized
- Security: input validation, no sensitive data in local storage
